## Critical Fixes Applied

1. **API Authentication** — Added `BAARA_API_KEY` middleware on all `/api/*` routes
2. **CORS Exact Allowlist** — Replaced `.includes()` with `Set.has()` exact origin matching
3. **CSP Header** — Added Content-Security-Policy, X-Frame-Options: DENY, X-Content-Type-Options: nosniff
4. **Rate Limiting** — 10 run/submit calls per minute per IP on execution endpoints
5. **Input Validation** — Clamped timeoutMs (max 1hr), maxRetries (max 10), maxBudgetUsd (max $10, default $2), validated executionType allowlist

## High Fixes Applied

1. **escapeHtml** — Added single-quote escaping (`'` → `&#x27;`)
2. **Unescaped fields** — `cronExpression`, `executionMode`, `job.id`, `job.attempt` now escaped in context-panel.js
3. **Error sanitization** — Global error handler returns opaque "Internal server error", full details logged server-side only
4. **permissionMode validation** — Enforced allowlist: default, acceptEdits, bypassPermissions
5. **agentConfig sanitization** — maxBudgetUsd ceiling ($10), maxTurns clamped (1-50)

## Accepted Risks (documented, not fixed)

- **raw_code execution**: Intentional feature for power users. Gated by API auth. Documented in CLAUDE.md.
- **bypassPermissions default**: Required for unattended scheduled execution. Protected by API auth.
- **Prompt injection**: Structural to LLM systems. Mitigated by auth (who can create tasks), budget caps, and tool scope.
- **PII in job output**: Calendar/email data stored in plaintext SQLite. Mitigated by API auth + localhost binding.
- **MCP server spawning**: Accepted risk for authenticated users. Future: allowlist permitted commands.
