## Architecture Decisions

1. **TypeScript over Rust**: Started as Rust experiment, switched to TypeScript for Agent SDK in-process integration (no subprocess bridge)
2. **bun:sqlite over better-sqlite3**: Bun doesn't support better-sqlite3's native bindings
3. **JSONL over SQLite for logs**: Append-only, streamable, jq-compatible, matches Claude Code's pattern
4. **~/.nexus/ over data/**: Proper data home separate from Claude Code's ~/.claude/
5. **Session tagging**: Tag Baara sessions with "baara" to filter out Claude Code sessions from listSessions()
6. **Chat-first over dashboard-first**: Primary interface is conversation, dashboard is secondary context panel
7. **In-process MCP server**: Tools call service functions directly, no subprocess or HTTP self-calls
8. **Direct mode default**: Bypasses queue for single-machine Mac mini usage

## Known Issues / Tech Debt

1. `as any` casts remain at Agent SDK boundary (query() options) — unavoidable without upstream types
2. Rate limiter keys on X-Forwarded-For (spoofable) — use getConnInfo for TCP address
3. `raw_code` execution has no sandbox — runs sh -c directly
4. Wasm and Docker isolation modes are UI-ready but not implemented
5. Distributed agent deployment (WebSocket phone-home) is designed but not built
6. Morning briefing task hasn't been tested end-to-end with real Gmail/Calendar
7. No automated CI/CD pipeline (tests run locally only)

## How to Run

```bash
cd ai-cowork/claude-task-executor
bun install
cp .env.example .env   # Default: subscription mode, no API key needed
bun start              # Server on http://0.0.0.0:3000
```

## How to Test

```bash
bunx playwright test               # All 39+ tests
bunx playwright test tests/us4*    # Single suite
bunx tsc --noEmit                  # Typecheck
```
